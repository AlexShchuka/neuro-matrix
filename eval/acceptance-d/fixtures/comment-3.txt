Q1 — цели / приоритеты / «никогда».
а) 3 задачи, которые система делает сама (не «улучшать себя»):
1) парсер-билдер — собрать и запустить веб-парсер через UI без ручного кода (наш реальный продукт, не самоцель);
2) self_improve над собственным кодом: задача → диф → тесты → мёрж, под гейтами;
3) сторож-самовосстановление: планировщик убивает зависшие задачи + периодические health-проверки.
б) приоритеты: надёжность > бюджет > скорость.
в) «никогда» (у нас живут как deontic-[F] антипаттерны + решения в decisions/):
— агенту никогда не пушить/мёржить в main напрямую (мёрж — отдельный контролируемый шаг);
— никогда не выпускать наружу секреты/IP/хосты (sanitization-gate);
— никогда не трогать чужой сетевой сегмент в доме (зафиксированное решение);
— никогда не коммитить в обход гейтов (--no-verify запрещён);
— никогда не принимать внешний совет без сверки с проектом (это ровно та роль, из которой пишу).
Про «eval чего»: поймали сами — у нас есть антипаттерн «валидация против суррогата вместо реальной цели». Целевая функция = парсер проходит реальную приёмку, а не зелёные тесты сами по себе.
