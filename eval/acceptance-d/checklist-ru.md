# Чек-лист: вынос кода в публичный репозиторий без секретов

Документ для Д. Прочитай перед первым публичным коммитом.

---

## 1. Переменные окружения и ключи

1. Убедись, что файл `.env` (или аналог) **не закоммичен** и добавлен в `.gitignore`.
2. В репозитории должен лежать только `.env.example` с именами переменных, но без значений:
   ```
   API_KEY=
   DATABASE_URL=
   SECRET_TOKEN=
   ```
3. Все реальные значения хранятся только на устройстве, **никогда** в репозитории.

---

## 2. Проверка на секреты перед коммитом — gitleaks

4. Установи [gitleaks](https://github.com/gitleaks/gitleaks) и добавь его в pre-commit хук:
   ```
   # .pre-commit-config.yaml
   repos:
     - repo: https://github.com/gitleaks/gitleaks
       rev: v8.18.2
       hooks:
         - id: gitleaks
   ```
5. Запускай `gitleaks detect --source . --verbose` перед каждым пушем, пока хук не настроен.
6. Если gitleaks найдёт секрет — **не пуши**, исправь сначала.

---

## 3. Защита на уровне GitHub

7. Включи **GitHub Push Protection** в настройках репозитория:
   `Settings → Code security → Push protection → Enable`.
   GitHub будет блокировать пуш, если обнаружит паттерн секрета.
8. Если секрет всё же попал в историю — **не удаляй только коммит** (это не скрывает его из истории Git). Нужна полная перезапись истории (`git filter-repo` или BFG Repo-Cleaner) + принудительный пуш. Предупреди коллег — их клоны устаревают.

---

## 4. Что ещё не должно попасть в публичный репо

9. IP-адреса, имена хостов, доменные имена внутренней инфраструктуры.
10. Логины, никнеймы, имена пользователей систем, к которым нет публичного доступа.
11. Пути к файлам вида `/home/username/...` или `/Users/name/...`.
12. Токены, пароли, сессионные куки — даже если уже истекли.
13. **Проверь не только последний коммит, но всю историю** (`git log -p | grep -i key` и аналоги).

---

## 5. Лицензия

14. Добавь файл `LICENSE` в корень репозитория.
15. Если не знаешь, какую лицензию выбрать — используй MIT (разрешительная, как у нас) или другую подходящую лицензию (например, Apache 2.0, AGPL).
16. Без `LICENSE` файла весь код автоматически закрыт по умолчанию — другие не могут его использовать легально.

---

## 6. Минимальная структура

17. `README.md` — что это, как запустить, что нужно настроить перед запуском.
18. `.gitignore` — включи `.env`, `__pycache__/`, `*.pyc`, `.DS_Store`, `node_modules/`.
19. `requirements.txt` или аналог — зафиксированные версии зависимостей.
20. `.env.example` — шаблон переменных без значений (см. п. 2).

---

## Быстрая проверка перед пушем

```bash
# 1. Нет ли .env в индексе?
git status | grep '\.env'

# 2. Нет ли секретов в истории?
gitleaks detect --source . --verbose

# 3. Есть ли LICENSE и README.md?
ls LICENSE README.md .env.example .gitignore
```

Если все три команды чистые — можно пушить.
