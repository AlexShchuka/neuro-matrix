neuro-matrix

An anti-“neuroslop” harness for Claude Code that keeps AI-assisted coding anchored to reality.

What is neuro-matrix?

neuro-matrix is a Claude Code plugin — a calibration protocol for an AI + developer co-system. It contributes runtime invariants (with a randomized per-turn self-check), deterministic hooks (cycle-detector, approval gate, verification gate), five agents (developer, analyzer, critic, epistemic-auditor, translator), and a held-out evaluation harness. The single goal threading every layer: judge every output by what is true and reproducible, never by what merely sounds plausible.

Install

/plugin marketplace add AlexShchuka/neuro-matrix
/plugin install neuro-matrix@neuro-matrix

Requires Claude Code.

Why it exists

AI-scaled development tends to accrete neuroslop — code that looks plausible but is architecturally meaningless. neuro-matrix is the anchor against that failure: hypothesis-only claims paired with tool evidence, per-mutation consent gating, halt-on-no-progress, mutual-doubt checks, and an approval + verification dual gate before anything lands. Designed under game theory (a Nash-style cooperative equilibrium) and systems theory.

How is it different from a prompt or a set of rules?

The protocol calibrates priors rather than listing imperative steps, and it is enforced at runtime by hooks and a sampled invariant self-check — not left to the model's memory. Both the AI and the developer are assumed fallible; the system is built to catch errors on both sides.
