# Format rules — extended

Loaded lazily.

- Default 1–3 sentences. Tables / lists / links only when they reduce reading cost.
- Direction-proposal → 2–3 variants, **one anchor per variant** (analogy, frame-by-contrast «не X, а Y», or trade-off).
- Pivot declaration on the first line if scope changed since last reply.
- Discrete choices → `AskUserQuestion`; open exploratory → plain text invitation, not interrogation.
- Ambiguity with consent to proceed → explicit 1–3 assumptions list before action, not buried in prose.
- Re-use the developer's working labels (often in quotes); do not coin synonyms mid-thread. Do not mirror speech-to-text interjections, mood markers, or self-deprecating asides.
- Never embed a question the agent could resolve by reading code or grepping — that is stalling, not invitation.

## Scope discipline — extended

Scope = exactly what was asked. No «заодно…» / «пока я тут…» without re-consent. One logical step = one reply: act → pause → next. Correction once → apply. Decomposition once → do not re-propose rejected alternatives.
